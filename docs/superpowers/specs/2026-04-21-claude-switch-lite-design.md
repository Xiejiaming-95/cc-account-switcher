# Claude 本地账号轻量切换工具设计文档

## 1. 目标

构建一个仅面向 Windows 本地环境的 Claude 账号轻量切换工具。

第一版只交付两个入口文件：
- `claude_switch.py`
- `claude_switch.bat`

工具负责保存和恢复本地 Claude 账号快照，不负责登录、订阅管理、token 刷新、进程管理或多平台兼容。

## 2. 设计结论

### 2.1 实现方式

采用单文件 Python 主程序方案，在 `claude_switch.py` 内按照职责拆分函数分组。

`claude_switch.bat` 保持极简，只负责调用 Python 脚本，不承载业务逻辑。

### 2.2 工具目录结构

```text
D:/work/ccswap/
├─ claude_switch.py
├─ claude_switch.bat
├─ accounts/
│  ├─ 主号/
│  │  ├─ credentials.json
│  │  ├─ config.json
│  │  └─ meta.json
│  └─ 备用号/
│     ├─ credentials.json
│     ├─ config.json
│     └─ meta.json
└─ docs/
   └─ superpowers/
      └─ specs/
         └─ 2026-04-21-claude-switch-lite-design.md
```

其中：
- `accounts/` 位于项目根目录，与两个入口文件同级
- 每个子目录对应一个账号存档
- 子目录名即备注名

## 3. 当前环境检测

### 3.1 Claude 本地文件路径

Windows 下按以下规则检测：

- 凭据文件：`%USERPROFILE%/.claude/.credentials.json`
- 配置文件优先级：
  1. `%USERPROFILE%/.claude/.config.json`
  2. `%USERPROFILE%/.claude.json`

实现时统一转换为绝对路径后使用。

### 3.2 当前账号识别

当前账号优先从配置文件读取：

- `oauthAccount.emailAddress`

识别规则：
- 识别成功：首页显示邮箱
- 识别失败：首页显示“未识别”

### 3.3 当前账号是否已保存

第一版采用简单邮箱匹配：

1. 读取当前识别出的邮箱
2. 遍历 `accounts/` 下各账号目录的 `meta.json`
3. 若某条 `detected_email` 与当前邮箱一致，则认定为已保存
4. 首页显示：
   - `已保存（备注名）`
   - `未保存`
   - `未知`

当当前邮箱无法识别时，首页状态直接显示“未知”。

## 4. 程序结构

`claude_switch.py` 保持单文件，但按职责组织为以下函数分组：

### 4.1 路径与环境函数

负责：
- 获取项目根目录下的 `accounts/` 路径
- 获取当前 Claude 配置文件路径
- 获取当前 Claude 凭据文件路径
- 读取当前配置中的邮箱

### 4.2 档案读写函数

负责：
- 保存当前账号档案
- 切换到指定账号档案
- 读取单个账号的 `meta.json`
- 枚举账号列表
- 删除指定账号目录

### 4.3 交互函数

负责：
- 渲染首页状态
- 显示主菜单
- 读取编号输入
- 读取备注名输入
- 执行二次确认

### 4.4 工具函数

负责：
- 当前时间格式化
- 备注名合法性校验
- 列表格式化展示

## 5. 首页与主菜单

启动后先展示当前环境状态，再展示固定主菜单：

```text
==============================
 Claude 本地账号切换工具
==============================
当前登录账号：abc@example.com
当前账号状态：已保存（主号）

1. 保存当前账号
2. 切换到已保存账号
3. 查看账号列表
4. 删除账号存档
5. 退出

请输入编号：
```

若无法识别当前账号，则显示：

```text
当前登录账号：未识别
当前账号状态：未知
```

默认交互规则：
- 完成“保存 / 切换 / 查看列表 / 删除”后，返回主菜单
- 所有用户可见文本均为中文
- 主菜单编号固定为 1-5

## 6. 账号备注名规则

备注名输入规则：
- 允许中文、英文、数字、空格
- 自动去除首尾空格
- 去除后若为空，提示重新输入
- 若包含 Windows 非法路径字符 `< > : " / \\ | ? *`，提示重新输入

默认备注名规则：
- 若识别出当前邮箱，则将邮箱作为默认备注名显示给用户
- 用户可以直接确认，也可以修改为更易识别的名称，例如“主号”或“工作号”
- 若未识别邮箱，则要求用户手动输入备注名

## 7. 核心流程

### 7.1 启动流程

程序启动后按以下顺序执行：

1. 定位当前 Claude 配置文件与凭据文件
2. 尝试读取 `oauthAccount.emailAddress`
3. 扫描 `accounts/` 下所有账号档案
4. 判断当前账号是否已保存
5. 展示首页状态
6. 显示主菜单

### 7.2 保存当前账号

执行顺序：

1. 检查当前 Claude 配置文件和凭据文件是否存在
2. 尝试识别当前邮箱
3. 生成默认备注名
4. 用户确认或修改备注名
5. 校验备注名合法性
6. 若目标账号目录已存在，执行二次确认
7. 复制当前 `credentials.json` 与配置文件为档案副本
8. 生成或更新 `meta.json`
9. 提示保存成功并返回主菜单

保存时 `meta.json` 字段定义为：

```json
{
  "name": "主号",
  "detected_email": "abc@example.com",
  "saved_at": "2026-04-21 20:30:00",
  "updated_at": "2026-04-21 20:30:00",
  "last_switched_at": "",
  "source_paths": {
    "credentials": "C:/Users/用户名/.claude/.credentials.json",
    "config": "C:/Users/用户名/.claude/.config.json"
  }
}
```

字段规则：
- 首次创建档案时写入 `saved_at`
- 每次保存都更新 `updated_at`
- 保存操作不修改 `last_switched_at`
- `detected_email` 识别不到时保存为空字符串

### 7.3 切换到已保存账号

执行顺序：

1. 列出所有已保存账号
2. 用户输入编号选择目标账号
3. 校验目标账号目录中 `credentials.json`、`config.json`、`meta.json` 是否齐全
4. 明确提示用户确认所有 Claude 相关窗口均已关闭
5. 只有用户明确确认后，才执行切换
6. 用目标账号档案覆盖当前 Claude 凭据文件与配置文件
7. 更新目标账号 `meta.json` 中的 `last_switched_at`
8. 提示切换完成，并提醒用户手动重新打开 Claude
9. 返回主菜单

第一版不做以下动作：
- 自动关闭 Claude 相关进程
- 自动重启 Claude
- 自动刷新 token

### 7.4 查看账号列表

列表至少显示：
- 编号
- 备注名
- 自动识别邮箱
- 最近更新时间
- 最近切换时间

展示规则：
- `last_switched_at` 为空时显示“未切换”
- `detected_email` 为空时显示“未识别邮箱”
- 单个账号异常不影响整体列表展示

对于损坏档案：
- 如果 `meta.json` 缺失、损坏或字段不完整，该项在列表中显示为“异常”或“信息不完整”
- 程序继续列出其他账号，不因单个异常中断

### 7.5 删除账号存档

执行顺序：

1. 列出所有已保存账号
2. 用户输入编号选择目标账号
3. 明确显示将删除的备注名
4. 执行二次确认
5. 删除对应账号目录
6. 提示删除完成
7. 返回主菜单

删除范围仅限目标账号档案目录，不删除：
- 当前真实 Claude 配置文件
- 当前真实 Claude 凭据文件
- 工具本体文件

## 8. 错误处理

第一版只处理必要错误，不做复杂恢复机制。

### 8.1 阻断型错误

遇到以下情况时，输出中文错误提示并返回主菜单：
- 找不到 Claude 配置文件
- 找不到 Claude 凭据文件
- 当前未检测到可保存的登录状态
- 目标账号缺少必要文件，无法切换
- 用户输入编号非法
- 备注名为空
- 备注名包含非法路径字符
- 用户取消覆盖、切换或删除

### 8.2 非阻断型错误

遇到以下情况时，不中断整个程序：
- 某个账号的 `meta.json` 损坏
- 某个账号的信息不完整

处理方式：
- 在列表中标记异常
- 继续显示其他可用账号

### 8.3 第一版明确不做

- 自动回滚事务
- 文件锁
- 并发保护
- 跨进程协调

## 9. `claude_switch.bat` 设计

`claude_switch.bat` 只负责：
- 调用 `python claude_switch.py`
- 保留窗口，避免双击执行后立即闪退

`claude_switch.bat` 不负责：
- 菜单逻辑
- 账号保存逻辑
- 账号切换逻辑
- 文件读写逻辑

## 10. 验证方案

在声称任务完成前，尽量覆盖三类验证。

### 10.1 格式检查

使用以下方式做最轻量语法校验：

```bash
python -m py_compile claude_switch.py
```

### 10.2 自动化检查

使用 Python 标准库 `unittest` 编写最小测试，重点覆盖不依赖真实本机 Claude 环境的逻辑，例如：
- 备注名合法性校验
- `meta.json` 生成与读取
- 当前账号是否已保存的判断
- 损坏档案不会导致列表逻辑崩溃

### 10.3 菜单级手工走查

使用临时目录模拟 `%USERPROFILE%` 下的 Claude 文件，手工验证：
- 启动
- 保存当前账号
- 查看账号列表
- 切换到已保存账号
- 删除账号存档

说明要求：
- 明确区分“模拟环境验证”和“真实本机 Claude 环境验证”
- 若某些场景未在真实环境验证，必须诚实说明

## 11. 本次实现范围结论

第一版实现以最小、直接、可维护为优先级，遵循以下原则：
- 只支持 Windows 本地环境
- 只依赖 Python 标准库
- 只交付 `claude_switch.py` 与 `claude_switch.bat`
- 不引入第三方依赖
- 不增加 GUI、加密、并发控制、进程管理等扩展能力

该设计足以覆盖以下需求：
- 保存当前账号
- 切换到已保存账号
- 查看账号列表
- 删除账号存档
- 启动时显示当前登录账号
- 启动时显示当前账号是否已保存
