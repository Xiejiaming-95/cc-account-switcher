# Claude 本地账号轻量切换工具需求文档

## 1. 项目目标

构建一个仅面向 Windows 本地环境的 Claude 账号轻量切换工具。

工具只使用以下两种交付形式：
- `claude_switch.py`
- `claude_switch.bat`

工具目标是提供一个足够轻、足够简单、足够稳定的本地账号快照切换方案，支持保存当前 Claude 登录状态、切换到已保存账号、查看账号列表、删除账号存档。

本工具不负责登录 Claude，不负责订阅管理，只负责在本地保存和恢复 Claude 当前账号状态。

---

## 2. 范围与非目标

### 2.1 第一版范围

第一版必须支持以下功能：
- 保存当前账号
- 切换到已保存账号
- 查看账号列表
- 删除账号存档
- 启动时显示当前登录账号
- 启动时显示当前账号是否已保存

### 2.2 明确不做的内容

第一版不包含以下内容：
- 图形界面
- 打包为 exe
- 自动关闭或重启 Claude 相关进程
- 自动刷新 token
- 凭据加密
- Windows Credential Manager 集成
- 云同步
- 多平台兼容
- VS Code 扩展专项适配
- 文件锁、事务回滚、并发保护

---

## 3. 用户使用方式

### 3.1 启动方式

用户通过双击 `claude_switch.bat` 启动工具。

### 3.2 入口交互

工具启动后，先展示当前环境状态，再展示菜单。

首页应显示：
- 当前登录账号
- 当前账号状态（已保存 / 未保存 / 未知）

示例：

```text
==============================
 Claude 本地账号切换工具
==============================
当前登录账号：abc@example.com
当前账号状态：已保存（主号）

1. 保存当前账号
2. 切换到已保存账号
3. 查看账号列表
4. 删除账号存档
5. 退出

请输入编号：
```

如果无法识别账号，则显示：

```text
当前登录账号：未识别
当前账号状态：未知
```

## 4. 目录结构

工具目录结构如下：

```text
工具目录/
├─ claude_switch.py
├─ claude_switch.bat
└─ accounts/
   ├─ 主号/
   │  ├─ credentials.json
   │  ├─ config.json
   │  └─ meta.json
   ├─ 备用号/
   │  ├─ credentials.json
   │  ├─ config.json
   │  └─ meta.json
```

说明：
- `accounts/` 位于脚本同目录
- 每个子目录对应一个账号存档
- 子目录名即用户备注名

## 5. 数据文件说明

### 5.1 `credentials.json`

保存当前 Claude 本地凭据快照。

用途：
- 切换时恢复目标账号的本地认证状态

### 5.2 `config.json`

保存当前 Claude 本地配置快照。

用途：
- 保存账号身份相关配置
- 切换时保持本地账号信息一致

### 5.3 `meta.json`

保存轻量元数据。

建议字段：

```json
{
  "name": "主号",
  "detected_email": "abc@example.com",
  "saved_at": "2026-04-21 20:30:00",
  "updated_at": "2026-04-21 20:30:00",
  "last_switched_at": "",
  "source_paths": {
    "credentials": "C:/Users/用户名/.claude/.credentials.json",
    "config": "C:/Users/用户名/.claude/.config.json"
  }
}
```

字段含义：
- `name`：备注名
- `detected_email`：自动识别出的邮箱，识别不到则为空字符串
- `saved_at`：首次保存时间
- `updated_at`：最近一次覆盖保存时间
- `last_switched_at`：最近一次切换到该账号的时间
- `source_paths`：保存时使用的源文件路径，便于后续排查

## 6. 菜单与交互流程

### 6.1 主菜单

菜单项固定为：

1. 保存当前账号
2. 切换到已保存账号
3. 查看账号列表
4. 删除账号存档
5. 退出

所有危险操作都必须二次确认。

### 6.2 保存当前账号

流程：
1. 读取当前 Claude 配置和凭据文件
2. 自动尝试识别当前邮箱
3. 将识别出的邮箱作为默认备注名显示
4. 允许用户修改备注名并确认
5. 若目录已存在，提示是否覆盖
6. 保存 `credentials.json`、`config.json`、`meta.json`
7. 提示保存成功

要求：
- 如果识别不到邮箱，仍允许用户手动命名后保存
- 如果备注名为空或非法，要求重新输入

### 6.3 切换到已保存账号

流程：
1. 列出所有已保存账号
2. 用户输入编号选择目标账号
3. 提示用户确认：已关闭所有 Claude 相关窗口
4. 只有用户明确确认后才继续执行
5. 用目标账号档案覆盖当前 Claude 凭据与配置
6. 更新 `meta.json` 中的 `last_switched_at`
7. 提示切换完成，并提醒用户手动重新打开 Claude

要求：
- 工具不自动关闭进程
- 工具不自动重启程序
- 工具只依赖用户确认后继续

### 6.4 查看账号列表

列表至少显示以下内容：
- 编号
- 备注名
- 自动识别邮箱
- 最近更新时间
- 最近切换时间（可为空）

示例：

```text
1. 主号    | abc@example.com  | 更新: 2026-04-21 18:20 | 最近切换: 2026-04-21 19:00
2. 工作号  | work@example.com | 更新: 2026-04-20 09:15 | 最近切换: 未切换
3. 测试号  | 未识别邮箱       | 更新: 2026-04-18 22:01 | 最近切换: 未切换
```

要求：
- 某个档案损坏时，不应导致整个列表崩溃
- 异常账号应在列表中标记为异常或信息不完整

### 6.5 删除账号存档

流程：
1. 列出所有已保存账号
2. 用户输入编号
3. 明确提示将删除的备注名
4. 要求二次确认
5. 删除对应账号目录
6. 提示删除完成

要求：
- 删除的是本地存档，不是当前 Claude 真实配置
- 不删除工具本体文件

## 7. 路径检测规则

### 7.1 设计原则

第一版采用固定优先级路径检测，不做复杂探测。

策略：
- 按预设候选路径依次检查
- 找到可用文件即采用
- 若找不到则明确报错，不做模糊猜测

### 7.2 Claude 本地路径候选

Windows 下优先尝试以下位置：

- `%USERPROFILE%/.claude/.credentials.json`
- `%USERPROFILE%/.claude/.config.json`
- `%USERPROFILE%/.claude.json`

实现时统一转换为绝对路径后使用。

### 7.3 配置文件选择规则

配置文件按以下顺序选取：
1. `%USERPROFILE%/.claude/.config.json`
2. `%USERPROFILE%/.claude.json`

找到第一个存在的文件后，将其认定为当前有效配置文件。

### 7.4 凭据文件选择规则

凭据文件优先采用：
1. `%USERPROFILE%/.claude/.credentials.json`

若不存在，则提示当前未检测到可切换的登录状态。

---

## 8. 当前账号识别规则

### 8.1 识别目标

第一版只要求尽量识别当前邮箱，不要求复杂账号指纹。

### 8.2 识别来源

优先从当前配置文件中读取以下字段：
- `oauthAccount.emailAddress`

### 8.3 识别结果处理

- 识别成功：
  - 首页显示当前邮箱
  - 保存时将邮箱作为默认备注名
  - 用于判断当前账号是否已保存
- 识别失败：
  - 首页显示“未识别”
  - 当前账号状态显示“未知”
  - 保存时允许用户手动输入备注名

---

## 9. 当前账号是否已保存的判断规则

第一版采用简单匹配规则：

1. 读取当前识别到的邮箱
2. 遍历 `accounts/` 下所有账号目录中的 `meta.json`
3. 若存在 `detected_email` 与当前邮箱一致的档案，则认定为已保存
4. 首页显示：
   - `已保存（备注名）`
   - `未保存`
   - `未知`

说明：
- 第一版不做组织信息匹配
- 第一版不做更复杂的指纹比较

---

## 10. 保存、切换与删除规则

### 10.1 保存规则

执行顺序：
1. 检查当前 Claude 配置文件和凭据文件是否存在
2. 尝试自动识别邮箱
3. 生成默认备注名
4. 用户确认或修改备注名
5. 创建账号目录
6. 保存配置与凭据副本
7. 生成或更新 `meta.json`
8. 如目录已存在，必须二次确认后才覆盖

### 10.2 切换规则

执行顺序：
1. 校验目标账号目录是否存在
2. 校验 `credentials.json`、`config.json`、`meta.json` 是否齐全
3. 提示用户确认已关闭所有 Claude 相关窗口
4. 用户确认后，覆盖当前 Claude 凭据文件
5. 覆盖当前 Claude 配置文件
6. 更新目标账号的 `last_switched_at`
7. 提示切换完成

### 10.3 删除规则

执行顺序：
1. 选择目标账号
2. 明确显示将删除的备注名
3. 用户二次确认
4. 删除对应目录
5. 提示删除成功

---

## 11. 错误处理边界

第一版只处理必要错误，不做复杂恢复机制。

### 11.1 必须处理的错误

- 找不到 Claude 配置文件
- 找不到 Claude 凭据文件
- 当前未检测到可保存的登录状态
- 账号目录缺少必要文件
- `meta.json` 损坏或格式错误
- 备注名为空
- 备注名包含非法路径字符
- 用户取消覆盖、切换或删除

### 11.2 错误处理原则

- 错误提示必须是中文且容易理解
- 尽量告诉用户下一步该怎么做
- 不因单个账号损坏导致整个工具崩溃

### 11.3 第一版明确不做

- 自动回滚事务
- 文件锁
- 并发保护
- token 刷新
- 跨进程协调

## 12. 最小技术约束

第一版技术约束如下：

- 运行环境：Windows + Python 3.x
- 只使用 Python 标准库
- 不依赖第三方包
- 不打包为 exe
- bat 文件只负责调用 Python 脚本并展示菜单入口

建议使用的标准库：
- `os`
- `json`
- `shutil`
- `pathlib`
- `datetime`

## 13. 验收标准

### 13.1 启动验收

- 双击 `claude_switch.bat` 能正常启动
- 首页能显示当前登录账号
- 首页能显示当前账号是否已保存
- 无法识别账号时，工具仍能继续使用

### 13.2 保存验收

- 已登录 Claude 时，能将当前状态保存到 `accounts/备注名/`
- 自动生成 `credentials.json`、`config.json`、`meta.json`
- 重名时会提示是否覆盖
- 覆盖后 `updated_at` 更新

### 13.3 列表验收

- 能列出所有已保存账号
- 每项至少显示备注名、邮箱、更新时间
- 单个档案损坏不会导致列表整体失败

### 13.4 切换验收

- 切换前必须要求用户确认已关闭所有 Claude 相关窗口
- 确认后才执行切换
- 切换后会更新当前本地配置和凭据
- `last_switched_at` 更新
- 完成后提示用户手动重新打开 Claude

### 13.5 删除验收

- 能删除指定账号目录
- 删除前必须二次确认
- 删除不影响当前 Claude 真实环境之外的工具文件

## 14. 典型使用场景

### 场景 1：保存主号

1. 用户先登录 Claude 主号
2. 双击 `claude_switch.bat`
3. 工具显示当前邮箱
4. 用户选择“保存当前账号”
5. 工具自动给出默认备注名
6. 用户改成“主号”并确认
7. 保存成功

### 场景 2：切换到备用号

1. 用户先关闭所有 Claude 相关窗口
2. 双击 `claude_switch.bat`
3. 选择“切换到已保存账号”
4. 选择目标编号
5. 确认已关闭窗口
6. 工具执行切换
7. 用户手动重新打开 Claude

### 场景 3：发现当前账号未保存

1. 工具首页显示当前登录账号
2. 当前账号状态显示“未保存”
3. 用户直接选择“保存当前账号”
4. 完成后首页再次进入时应显示为“已保存”

## 16. 最终结论

本工具第一版定位为一个本地轻量可维护的 Claude 账号快照切换器。

它通过保存当前 Claude 的配置与凭据快照，在本地账号档案之间进行恢复切换，实现多账号快速切换。整体设计优先保证：
- 轻量
- 简单
- 中文可读
- 易于手工维护
- 出问题时容易排查
